This code file represents a simplified car park management system that tracks vehicles entering and exiting a car park, monitors temperature readings, and displays live information through a graphical interface. The program is built using several imported modules that provide the tools needed for timing, graphical interfaces, background processing, and structured program design. The first import, import threading, allows the program to run tasks concurrently. Normally, Python executes code line by line in a single sequence, but threading allows a separate task to run in the background while the main program continues operating. In this project, threading is used so the display can update continuously without freezing the graphical interface. For example, the code updater = threading.Thread(target=self.check_updates) creates a background thread that runs the check_updates method, and updater.start() begins executing that method separately from the main program. This thread repeatedly checks for new data and refreshes the display while the user continues interacting with the interface.

Another important module imported is import time, which provides functions related to time and delays. The program uses this module to record when cars enter and exit the car park, display the current time on the screen, and pause certain operations so they run at controlled intervals. For instance, when a car object is created the entry time is recorded using self.entry_time = time.localtime(), which stores the current system time. When a car leaves the car park, the method self.exit_time = time.localtime() records the exit time. The program also uses time.sleep(1) inside the display update loop to pause the thread for one second between updates. Another use of the time module appears in the logging system where timestamps are formatted for readability, such as timestamp = time.strftime("%Y-%m-%d %H:%M:%S", time.localtime()), which produces a formatted date and time like 2026-03-09 14:22:10.

The Car class represents a single vehicle parked in the car park. When a new Car object is created using car = Car(license_plate), it stores the vehicle’s license plate number and records the time the car entered the car park. This happens in the constructor where the line self.entry_time = time.localtime() saves the current time. Initially the exit time is set to None, indicating that the car is still parked. When the car leaves, the method leave() is called and the line self.exit_time = time.localtime() records the departure time. This simple class allows the system to store and track information about each car individually.

The CarparkManager class is the central component of the system and acts as the controller that manages all car park operations. It stores the total number of spaces using the class constant TOTAL_SPACES = 50, meaning the car park can hold fifty vehicles at maximum capacity. When the manager is initialized, it sets self._available_spaces = self.total_spaces so that initially all spaces are free. The manager also maintains a dictionary called self.cars = {}, which stores the cars currently parked in the car park. Each car is stored using its license plate as the key, such as self.cars[license_plate] = car, allowing the program to quickly find or remove cars as they enter or exit.

When a car arrives, the method incoming_car processes the event. The method first checks whether the license plate is empty using if not license_plate: return, ensuring invalid input is ignored. It also prevents duplicate entries by checking if license_plate in self.cars: return. If the car park is full, the condition if self._available_spaces <= 0: prevents additional cars from entering. If all checks pass, a new car object is created with car = Car(license_plate) and stored in the dictionary. The available space count is then decreased using self._available_spaces -= 1, and the event is recorded in the log system using self.log("Car entered", license_plate).

When a car leaves the car park, the method outgoing_car handles the process. It first verifies that the license plate exists in the dictionary using if license_plate not in self.cars:. If the car is not recognized, a log entry is created with self.log("Unknown car exit", license_plate). Otherwise, the car is removed from the dictionary using car = self.cars.pop(license_plate), and its exit time is recorded by calling car.leave(). The available spaces counter is then increased with self._available_spaces += 1, and the system logs the event with self.log("Car exited", license_plate).
